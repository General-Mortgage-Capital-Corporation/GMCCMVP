**2. Add new section** -- After the "API Functions" section and before the "UI Functions" section, add a new section:

```
// =============================================================================
// Program Matching Functions
// =============================================================================
```

In this section, add:

**`startMatching(listings)`** -- Fires parallel POST /api/match calls for each listing. For each listing:
- Set `listing.matchLoading = true`
- `fetch('/api/match', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(listing) })`
- On success (response.success is true): set `listing.matchData = data`, call `updateCardBadge(index, data.eligible_count)`
- On error (catch): silent failure -- no badge shown
- In `finally`: set `listing.matchLoading = false`, decrement `matchPending`, if `matchPending <= 0` call `onAllMatchesComplete()`
- Initialize `matchPending = listings.length` at the top of the function

**`updateCardBadge(index, eligibleCount)`** -- Finds the card by `document.querySelector('[data-index="' + index + '"]')`. Finds the `.property-badges` container within it. Removes the `.badge-programs-loading` skeleton if it exists. If `eligibleCount > 0`, creates a `<span>` with `className = 'badge badge-programs'` and `textContent` of `${eligibleCount} Program${eligibleCount !== 1 ? 's' : ''}` and appends it to badges.

**`onAllMatchesComplete()`** -- This is a placeholder that Plan 02 will extend. For now, just log: `// Filter bar population handled in Plan 02`

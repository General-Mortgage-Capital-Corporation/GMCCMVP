  <name>Task 1: ChromaDB vector store module, CLI entry point, and tests</name>
  <files>rag/vectorstore.py, rag/ingest.py, rag/__main__.py, tests/test_vectorstore.py, tests/test_ingestion.py</files>
  <behavior>
    - GeminiEmbeddingFunction.__call__ returns list of float lists (embeddings) for input documents
    - get_collection returns a ChromaDB collection object
    - store_chunks stores page chunks with program_name metadata and generates IDs like "{program_name}_page_{i}"
    - store_chunks is idempotent: calling twice with same program produces same result (deletes existing, re-adds)
    - query_program_info returns relevant chunks for a query string
    - query_program_info with program_name filter returns only that program's chunks
    - CLI ingest command processes PDFs from data/guidelines/ and writes JSON to data/programs/
    - CLI --review-only flag writes JSON but skips ChromaDB indexing
    - CLI --program flag processes only the named program
    - Re-running ingest produces identical JSON output (idempotent)
  </behavior>
  <action>
  1. Create `rag/vectorstore.py`:

     `GeminiEmbeddingFunction(EmbeddingFunction[Documents])`:
     - `__init__(self, api_key: str | None = None, model: str = EMBEDDING_MODEL)`:
       Creates `genai.Client(api_key=api_key)` if api_key provided, else `genai.Client()`.
       Stores model name. Uses EMBEDDING_MODEL from config as default.
     - `__call__(self, input: Documents) -> Embeddings`:
       Calls `self._client.models.embed_content(model=self._model, contents=input, config=types.EmbedContentConfig(task_type="RETRIEVAL_DOCUMENT", output_dimensionality=EMBEDDING_DIMENSIONS))`.
       Returns `[e.values for e in result.embeddings]`.

     `get_collection(chroma_dir: str = CHROMA_DIR) -> chromadb.Collection`:
     - Creates `chromadb.PersistentClient(path=chroma_dir)`.
     - Returns `client.get_or_create_collection(name=CHROMA_COLLECTION_NAME, embedding_function=GeminiEmbeddingFunction())`.

     `store_chunks(program_name: str, pages: list[dict], collection=None)`:
     - If collection is None, calls get_collection().
     - Deletes existing chunks for this program: `collection.get(where={"program_name": program_name})` then `collection.delete(ids=...)` if any exist.
     - For each page, creates ID `f"{program_name}_page_{i}"` and adds with metadata `{"program_name": program_name, "page_number": i + 1}`.
     - Uses `collection.add(ids=..., documents=..., metadatas=...)`.

     `query_program_info(query: str, program_name: str | None = None, n_results: int = 3, collection=None) -> dict`:
     - If collection is None, calls get_collection().
     - Builds kwargs with query_texts and n_results.
     - If program_name, adds `where={"program_name": program_name}`.
     - Returns `collection.query(**kwargs)`.

  2. Create `rag/ingest.py`:
     - Uses click for CLI interface.
     - `@click.command()` with options:
       - `--review-only` (flag): Extract and write JSON but skip ChromaDB indexing
       - `--program` (str, optional): Process only this program folder name
     - `def ingest(review_only, program)`:
       - Creates PROGRAMS_DIR with `os.makedirs(PROGRAMS_DIR, exist_ok=True)`.
       - Lists program directories: if --program specified, uses just that one; otherwise sorts all subdirs of GUIDELINES_DIR.
       - For each program directory:
         a. Finds .pdf files in the directory.
         b. Skips with warning if no PDF found.
         c. Stage 1: `pages = extract_pdf_to_markdown(pdf_path)`
         d. Concatenates: `full_md = "\n\n".join(p["text"] for p in pages)`
         e. Stage 2: `rules = structure_with_llm(full_md, prog_name)`
         f. Writes JSON: `data/programs/{program_name_lower}.json` via `json.dump(rules.model_dump(), f, indent=2)`
         g. Stage 3 (unless --review-only): `store_chunks(prog_name, pages)`
         h. Prints progress with click.echo.
       - Prints "Done." at end.

  3. Create `rag/__main__.py`:
     ```python
     from rag.ingest import ingest
     ingest()
     ```
     This enables `python -m rag` as the entry point.

     - Test query_program_info with program_name filter.
       Store chunks for two programs, query with filter, assert only filtered program returned.
     - Mark tests that require GEMINI_API_KEY with `@pytest.mark.integration`.

  5. Create `tests/test_ingestion.py`:

     - Test idempotent re-ingestion: run ingest twice, verify JSON output is identical.
     - Mark tests that call real APIs with `@pytest.mark.integration`.

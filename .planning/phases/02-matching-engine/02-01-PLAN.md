<objective>
Build the core deterministic matching engine that checks property listings against all GMCC program tiers and returns structured per-criterion eligibility results.

Purpose: This is the central logic for Phase 2 -- given a RentCast listing, determine which program tiers are Eligible, Potentially Eligible, or Ineligible based on deterministic rule checking (no LLM). The three-value logic (pass/fail/unverified) ensures missing listing data never causes false exclusions.

Output: `matching/` Python package with Pydantic models, property type mapping, FCC geocode fallback, and the `match_listing()` function. Full unit test coverage in `tests/test_matching.py`.
</objective>

  <action>
    Create `matching/matcher.py` with:

    **load_programs() -> list[ProgramRules]:**
    - Read all `.json` files from `rag.config.PROGRAMS_DIR`
    - Parse each with `ProgramRules.model_validate(data)`
    - Cache with `@lru_cache(maxsize=1)` (wrap in a tuple or use a module-level cache variable since lru_cache needs hashable args)
    - Return list of ProgramRules

    **Criterion check functions** (each returns CriterionResult):
    - `check_property_type(listing_type: str | None, tier: EligibilityTier) -> CriterionResult`: Map listing_type via RENTCAST_TO_PROGRAM. None or unmapped -> UNVERIFIED. Mapped value in tier.property_types -> PASS. Otherwise FAIL.
    - `check_loan_amount(price: float | None, tier: EligibilityTier) -> CriterionResult`: None -> UNVERIFIED. If price < tier.min_loan_amount -> FAIL (price is ceiling for loan, can't reach min). If tier has no min/max constraints -> PASS. Otherwise PASS (price >= min means a loan in range is possible; price > max is OK because with down payment the loan can be <= max).
    - `check_location(county: str | None, state: str | None, lat: float | None, lon: float | None, tier: EligibilityTier) -> CriterionResult`: If tier.location_restrictions is empty -> PASS (no restrictions). If county or state available, check against restrictions (substring match for county names, exact match for state codes). If no location data and no lat/lon -> UNVERIFIED. If lat/lon available but no county, call get_county_from_coordinates as fallback; if fallback fails -> UNVERIFIED.
    - `check_unit_count(listing_type: str | None, tier: EligibilityTier) -> CriterionResult`: If tier.unit_count_limits is empty -> PASS (no restriction). Infer units from PROPERTY_TYPE_UNITS. None inference -> UNVERIFIED. Inferred units in tier.unit_count_limits -> PASS. Otherwise FAIL.

    **match_tier(listing: ListingInput, tier: EligibilityTier) -> TierResult:**
    - Run all 4 criterion checks
    - Compute status: any FAIL -> INELIGIBLE, any UNVERIFIED (no FAIL) -> POTENTIALLY_ELIGIBLE, all PASS -> ELIGIBLE
    - Return TierResult with all criteria

    **match_listing(listing: ListingInput) -> list[ProgramResult]:**
    - Load programs via load_programs()
    - For each program, filter tiers to only those with "Purchase" in transaction_types (locked decision)
    - Skip occupancy filtering (locked decision: show all occupancy tiers)
    - Run match_tier for each qualifying tier
    - Collect matching_tiers = tiers where status != INELIGIBLE
    - Determine program status: best status across matching_tiers (ELIGIBLE > POTENTIALLY_ELIGIBLE). If no matching tiers, status = INELIGIBLE
    - Set best_tier = first ELIGIBLE tier name, or first POTENTIALLY_ELIGIBLE if none ELIGIBLE, or None
    - Return list of ProgramResult (one per program)

    **matching/__init__.py:** Re-export key names: `from matching.matcher import load_programs, match_listing` and `from matching.models import *`
